# CLAUDE.md - atrice landing

Лендинг студии atrice. Структура: `backend/` (FastAPI + SQLite), `frontend/` (React + Vite + TypeScript), `shared/seed_content.json` (единый источник текстов), деплой на VPS через GitHub Actions.

## AI-маркеры - запрещены

В коде, комментариях, документации, коммитах и финальных ответах не использовать:

- длинное тире;
- "умные" кавычки (елочки и типографские);
- AI-клише и цепочки стрелок в прозе.

Вместо тире - дефис или перестроить фразу. Кавычки - прямые.

Правило действует и на продуктовые тексты (`shared/seed_content.json` и строки в БД): с 2026-07-07 тире и елочки там тоже не используются (решение пользователя, миграция 002_strip_ai_markers). Карта замен в самой миграции - единственное легитимное место этих символов в коде.

## Коммиты и ветки

- Формат: `<type>(<scope>): <тезисно, императив, маленькая буква, без точки>`. Ровно одна строка.
- `<type>`: `feat` / `fix` / `refactoring` / `docs` / `ci` / `chore`.
- `<scope>`: `backend`, `frontend`, `content`, `infra`, `docs`.
- Без `Co-Authored-By` и "generated with"-плашек. Без многострочных тел: подробности в описании PR или в ответе пользователю.
- Рабочая ветка `main`. Пуш в `main` = автодеплой на VPS после зеленого CI, поэтому пушить только проверенное. Для рискованных изменений - отдельная ветка и PR.

## Тесты

Новые тесты не писать без явной просьбы пользователя. Существующие backend-тесты (`backend/tests/`, pytest) не удалять и прогонять перед коммитом.

## Соглашения по коду

Backend:
- ruff, line-length 100; прогонять перед коммитом (`.venv/bin/ruff check .`).
- Докстринги однострочные и только там, где имя не передает смысл.
- Инлайн-комментарии - только когда код нельзя сделать понятнее переименованием.
- Импорты на уровне модуля.

Frontend:
- TypeScript без `any` и `@ts-ignore` без обоснования.
- Никаких новых UI-библиотек и state-managers: только react, react-dom, react-router-dom.
- Проверка перед коммитом: `npx tsc --noEmit -p tsconfig.app.json && npm run build`.

Общее: паттерны вводить только под реальную проблему (YAGNI). Тихих рефакторингов "по пути" не делать: нашел проблемное место вне задачи - показать пользователю и предложить варианты.

## Данные и миграции

- Все тексты лендинга лежат в SQLite и правятся через `/admin`.
- `shared/seed_content.json` наполняет только пустую базу.
- Изменение существующих данных - через `backend/app/migrations.py`: пара (id, функция), применяются на старте по одному разу, применённые фиксируются в таблице `applied_migrations`. Миграция обязана быть идемпотентной.

## Self-review перед "готово"

Перед финальным "готово" явно перечислить пункты с пометками `[x]` (прошел) / `[!]` (нарушение, нужно решение пользователя) / `[-]` (неприменимо, объяснить):

- AI-маркеры: чисто (код, доки, коммиты, ответ);
- формат коммитов соблюден;
- pytest: passed;
- ruff: clean;
- tsc + vite build: без ошибок;
- новых тестов не добавлено (или добавлено по явной просьбе);
- если пушил в main: CI и Deploy зеленые, прод отвечает.

Молча пройденный чек-лист = не пройденный.

## Окружение разработки

- Активное pip-зеркало из `~/.config/pip/pip.conf` (nirtest-portal) сломано. Рабочее: `https://nexus-omg.pak-cspmz.ru/repository/iro-pypi-group/simple/` с сертификатом из `~/.pip/pip.conf`.
- npm использует публичный registry, работает.
