# atrice: Telegram-бот студии

Дата: 2026-07-07
Статус: одобрено пользователем (вариант A, MVP)

## Цель

Бот с полноценным меню: презентует студию (портфолио, услуги, о студии) и
принимает заявки анкетой. Заявки попадают владельцу в личку и в базу, в
админке появляется раздел для работы с ними. Контент бота управляется из
админки без релизов.

## Решения (зафиксированы с пользователем)

| Вопрос | Решение |
|---|---|
| Назначение | Меню: портфолио, услуги, заявка, о студии |
| Заявки | В личку владельцу + в SQLite, раздел в админке |
| Процесс | Отдельный сервис на aiogram 3, long polling (webhook позже настройкой) |
| Доступ к данным | Бот не пишет в SQLite напрямую: заявки через POST /api/leads с сервисным токеном, контент через публичные GET |
| Контент бота | В базе, правится в существующей админке; картинки грузятся через админку |
| Язык бота | Русский (i18n позже при необходимости) |

## Структура

```
bot/
├── app/
│   ├── main.py          # запуск: Bot, Dispatcher, polling
│   ├── config.py        # env: BOT_TOKEN, OWNER_CHAT_ID, BACKEND_URL, SERVICE_TOKEN
│   ├── backend.py       # HTTP-клиент к бэкенду (content, portfolio, leads)
│   ├── menu.py          # /start, главное меню, услуги, о студии
│   ├── portfolio.py     # карточки портфолио с листанием
│   └── lead.py          # FSM-анкета заявки
├── pyproject.toml       # aiogram>=3, httpx; dev: ruff
└── Dockerfile
```

Бэкенд дополняется файлами `app/leads.py` (роуты заявок + проверка
сервисного токена) и `app/portfolio.py` (роуты портфолио + сохранение
файлов в data/uploads; раздача /uploads монтируется в main.py). Фронтенд-
админка: компоненты `admin/Leads.tsx` и `admin/Portfolio.tsx`, подключаются
вкладками в существующей странице Admin.

## Меню и сценарии бота

- `/start`: обложка (картинка) + inline-меню: Портфолио, Услуги,
  Оставить заявку, О студии.
- Портфолио: карточки (фото + заголовок + текст) с кнопками "назад/вперёд"
  и возвратом в меню. Пустое портфолио: дружелюбная заглушка.
- Услуги: карточки из существующих ключей content svc1..svc4 (единый
  источник с сайтом).
- О студии: aboutP1 + aboutP2 + кнопка-ссылка на https://atrice.ru.
- Заявка (FSM, 3 шага): 1) задача свободным текстом, 2) тип проекта
  кнопками (MVP / лендинг / веб-приложение / поддержка), 3) желаемые
  сроки. Подтверждение перед отправкой, отмена на любом шаге. Имя и
  username берутся из Telegram.
- После отправки: сохранение через API + сводка владельцу в личку +
  благодарность клиенту с обещанием ответа в течение дня.

Тексты всех реплик бота: новые ключи в таблице content, секция "Бот"
(например botWelcome, botMenuPortfolio, botLeadAskTask, botLeadThanks и
т.д.), редактируются в текущей админке. Бот кэширует контент на короткое
время (1-5 минут) и переживает недоступность бэкенда: сообщает "попробуйте
позже", не падает.

## API бэкенда

- `POST /api/leads` (заголовок X-Service-Token == env SERVICE_TOKEN):
  создать заявку. 401 при неверном токене.
- `GET /api/leads` (админ-сессия): список, новые сверху.
- `PATCH /api/leads/{id}` (админ-сессия): смена статуса.
- `GET /api/portfolio` (публичный): включённые карточки по порядку,
  image_path в виде абсолютного URL.
- `POST /api/portfolio` (админ, multipart: title, text, image):
  создать карточку; файл сохраняется в data/uploads с безопасным именем.
- `PUT /api/portfolio/{id}` (админ): правка полей, опционально новая
  картинка; `DELETE /api/portfolio/{id}` (админ): удаление вместе с файлом.
- Статика `/uploads/{file}`: раздача каталога data/uploads.
- IP-гейт админки распространяется на новые админские эндпоинты
  автоматически (мутирующие методы /api/* уже под гейтом; GET /api/leads
  добавить в защищаемые пути).

Сид-ключи бота добавляются в shared/seed_content.json (секция "Бот"),
для существующих баз - миграция 007 с этими ключами.

## Админка

- Раздел "Заявки": таблица (дата, имя, username со ссылкой t.me, тип,
  сроки, задача, статус), кнопка "Обработана"/"Вернуть в новые".
- Раздел "Портфолио": список карточек с превью картинки, добавление
  (заголовок, текст, файл), правка, удаление, порядок через поле position.
- Тексты бота появляются в существующем списке текстов автоматически
  (новая секция "Бот").

## Инфраструктура

- docker-compose: сервис `bot` (build bot/, env_file .env, restart
  unless-stopped, зависит от app). Внешних портов у бота нет.
- .env: добавляются BOT_TOKEN, OWNER_CHAT_ID, SERVICE_TOKEN
  (BACKEND_URL по умолчанию http://app:8000 внутри сети compose).
- CI: ruff для bot/. Деплой без изменений (compose up пересобирает оба).
- Volume data/ уже проброшен: uploads живут в data/uploads.

## Обработка ошибок

- Бэкенд недоступен: бот отвечает "Сервис временно недоступен, попробуйте
  позже" и логирует; анкета не теряется до подтверждения (состояние FSM).
- Ошибка отправки владельцу: заявка всё равно сохранена в базе; лог.
- Невалидный SERVICE_TOKEN: 401, бот сообщает пользователю об ошибке.

## Вне скоупа (вариант B, позже)

- Webhook-транспорт, двуязычность бота, уведомления о деплоях/мониторинг,
  очередь недоставленных заявок, FSM-персистентность между рестартами.

## Проверка (без новых автотестов, по правилу проекта)

Живой прогон: /start, все разделы меню, листание портфолио, полная анкета
с подтверждением, заявка в личке владельца и в админке, смена статуса,
загрузка карточки портфолио с картинкой и её появление в боте.
